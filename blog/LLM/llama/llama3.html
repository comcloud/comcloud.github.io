Llama 3 & 3.1 概述

Llama 3是Meta于2024年4月推出的新一代开源大语言模型，作为Llama系列的最新成员，它在模型规模、多语言支持和推理能力等方面都有显著提升。随后发布的Llama 3.1进一步优化了训练数据和上下文长度，巩固了Llama系列在开源大模型领域的领先地位。

Llama 3系列的主要突破

  • 扩展的词汇量：128K token词汇表，支持更多语言和符号
  • 优化的分词器：提升编码效率15%，减少token数量
  • 多语言支持：显著改善非英语语言表现
  • 推理能力：数学和代码生成能力大幅提升
  • Llama 3.1增强：优化数据质量，扩展上下文长度

历史意义

Llama 3系列的发布标志着开源大语言模型在多语言和长上下文方面的重要突破：

  • 多语言标杆：首个在非英语语言上达到商用水平的开源大模型
  • 技术演进：证明了数据质量比数量更重要
  • 生态影响：催生大量多语言应用和衍生模型
  • Llama 3.1：回应社区需求，优化关键能力

核心架构与技术特点

Llama 3在Llama 2架构基础上进行了多项创新，这些改进共同提升了模型的多语言能力和计算效率。

Transformer架构优化

Llama 3保留了Llama 2的纯解码器Transformer架构，但进行了以下关键改进：

🔤

扩展词汇表

词汇表从32K扩展到128K token，显著提升编码效率，减少非英语语言的token数量。

🧮

优化分词器

新的分词算法提升15%的编码效率，特别改善代码和数学符号的处理。

🌐

多语言优化

架构调整和训练数据平衡显著提升非英语语言的理解和生成能力。

分组查询注意力(GQA)增强

Llama 3延续了Llama 2的GQA设计，但进行了以下优化：

GQA增强特性

  • 动态分组：根据输入长度动态调整分组策略
  • 内存优化：进一步减少KV缓存内存占用
  • 长上下文支持：优化长序列的注意力计算

旋转位置编码(RoPE)改进

Llama 3对RoPE进行了针对性优化以支持更长上下文：

RoPE(xₘ, m) = (Wₙxₙ)eⁱmθₙ
其中θₙ = 10000⁻²ⁿ/d，d是维度，n是位置索引

Llama 3.1进一步扩展了上下文长度支持，通过调整θₙ的基数和插值策略实现更稳定的长序列处理。

SwiGLU激活函数

Llama 3继续使用SwiGLU激活函数，其数学表达式为：

SwiGLU(x) = (swish(W₁x)) ⊙ (W₂x)
其中swish(x) = x · σ(x)，σ表示sigmoid函数，⊙表示逐元素乘法

实验表明，SwiGLU在多语言场景下比标准FFN具有更强的表达能力。

多语言能力突破

Llama 3在多语言支持方面取得了显著进展，使其成为首个在非英语语言上达到商用水平的开源大模型。

多语言训练数据

Llama 3使用了更平衡的多语言数据分布：

  • 英语：占比约50%，质量严格筛选
  • 欧洲语言：法语、德语、西班牙语等各占5-8%
  • 亚洲语言：中文、日语、韩语等各占3-5%
  • 其他语言：涵盖30+种语言的代表性文本

分词器优化

Llama 3的128K词汇表分词器针对多语言进行了特别设计：

词汇扩展
  • 包含更多非英语字符和子词
  • 优化罕见语言的分词效率
  • 平衡不同语言的token数量
编码效率
  • 中文token数量减少40%
  • 日语分词效率提升35%
  • 非拉丁文字支持显著改善
特殊符号
  • 数学符号全覆盖
  • 编程语言token优化
  • 表情符号支持

训练方法与数据处理

Llama 3的训练流程体现了"质量优于数量"的理念，通过精心设计的数据策略和训练技术实现了高效学习。

训练数据构成

Llama 3使用比前代更高质量和多样性的数据源：

数据收集

从公开可用的高质量来源收集数据，包括学术论文、技术文档、多语言书籍和经过筛选的网页内容。

数据过滤

使用多阶段过滤流程去除低质量内容，包括基于规则的过滤和分类器评分。

多语言平衡

精心平衡不同语言的数据比例，确保模型不会过度偏向英语。

Llama 3.1优化

进一步筛选训练数据，提升专业领域和低资源语言的数据质量。

预训练技术

Llama 3的预训练采用以下关键技术：

高效训练策略

  • 优化器：AdamW，β₁=0.9，β₂=0.95，ε=10⁻⁵
  • 学习率：余弦衰减，峰值2.5×10⁻⁴，最终降至峰值5%
  • 批量大小：动态调整，最大500万token
  • 权重衰减：0.1，梯度裁剪阈值1.0
